# 外部数据接入与深度利用实践作业

**课程**：智能应用开发  

---

## 任务

在你的智能助手项目中，接入一个外部数据源，实现**从数据到行动的完整闭环**。

---

## 需求一：数据接入与展示 [L1]（40分）

**要求**：

1. 接入一个与你的助手定位相关的外部数据源（API推送、定时拉取、文件导入或图片识别均可）
2. 数据存储到本地，能持久化
3. 在助手界面中展示这些数据（看板、对话、或任意方式）

**交付物**：

- 数据接入的核心代码
- 至少5条测试数据（正常、异常、边界情况）
- 展示效果的截图

---

## 需求二：数据驱动的智能响应 [L2-L3]

实现2个以上的数据智能应用：

### L2：异常检测与主动通知（30分）

- 设计至少2条异常检测规则（阈值、变化率、连续趋势等）
- 当异常发生时，系统主动通知用户
- 通知内容包含具体数值和具体建议

### L3：数据驱动的建议调整（30分）

- 设计至少2条联动规则（数据变化 → 建议/配置改变）目标
- 系统能根据数据自动调整行为
- 能验证调整的效果

**交付物**：

- 异常检测/联动规则的设计说明
- 核心代码
- 触发效果演示（截图）

---

## 加分项

| 项目                | 分值  |
| ----------------- | --- |
| 实现双向（异常检测 + 建议调整） | +10 |
| 趋势分析（连续数据的变化检测）   | +5  |
| 四级降级数据获取方案        | +5  |
| 自动生成周报/日报         | +5  |

---

## 评分标准

| 等级  | 分数      | 标准             |
| --- | ------- | -------------- |
| 优秀  | 90-100+ | L1完整，L2/L3有亮点  |
| 良好  | 75-89   | L1完整，L2/L3基本完成 |
| 中等  | 60-74   | L1完整，L2有尝试     |
| 及格  | 50-59   | L1完成，有接入但无智能响应 |
| 不及格 | <50     | 未接入外部数据        |

---

## 提交

1. **说明文档**（1-2页）：数据源选择、数据流设计、检测/联动规则
2. **演示**：截图（1-3分钟）

---

## 参考思路

| 助手类型 | 可接入数据源     | 智能响应方向        |
| ---- | ---------- | ------------- |
| 健康助手 | 体重秤/血压计/手表 | 指标异常预警、运动建议调整 |
| 学习助手 | 教务系统/番茄钟   | 成绩下滑预警、学习计划调整 |
| 理财助手 | 行情API/银行流水 | 亏损预警、消费异常提醒   |
| 旅行助手 | 航班/天气API   | 延误提醒、行程调整建议   |
| 任何助手 | 气象API      | 天气影响相关建议      |

没有合适的数据源？

**可以模拟一个——写个脚本定时生成数据POST到你的后端。**

可控，方便测试。
