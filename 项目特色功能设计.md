# 项目特色功能设计

---

## 一、目标人群画像

**目标人群**：50-80岁中老年人，尤其是65岁以上的银发群体。

**核心场景**：
- **居家养老**：子女不在身边，独自生活或与配偶同住
- **慢性病管理**：患有高血压、糖尿病、冠心病等慢性疾病
- **用药管理**：每天需要按时服用多种药物
- **健康监测**：定期测量血压、血糖、体重等指标

**关键痛点**：
1. **用药遗忘**：每天多种药物，容易漏服或错服
2. **健康数据孤立**：测量的数据无法有效分析和利用
3. **子女担忧**：子女无法实时了解父母健康状况
4. **紧急情况**：突发疾病时无法及时联系家人
5. **操作困难**：对智能手机操作不熟悉，界面复杂难以使用

**与年轻人的关键差异**：
- 不是简单记录数据，而是需要从测量到提醒到分析的完整健康管理链路
- 不是独自管理，而是需要与子女端形成闭环的远程关怀体系
- 不是追求功能丰富，而是需要简洁、易懂、语音化的交互方式

## 二、通用助手的不足

### 通用方案做了什么 vs 做不好什么

| 通用方案 | 能做什么 | 做不好什么 |
|---------|---------|-----------|
| **健康记录App**（如薄荷健康） | 记录饮食、计算卡路里 | 只能记，不能结合个人健康状况给出针对性建议 |
| **用药提醒App**（如用药助手） | 设置用药提醒 | 只管提醒，不管是否按时服用，无法追踪依从性 |
| **智能手表/手环** | 监测心率、步数 | 数据孤岛，无法与用药、饮食形成关联分析 |
| **通用对话助手**（如ChatGPT） | 回答健康问题 | 没有用户的真实健康数据，建议空泛，无法落地 |
| **视频通话**（如微信） | 与子女沟通 | 无法主动推送健康异常，子女需主动询问 |

### 关键差距分析

1. **数据割裂**：健康监测、用药管理、饮食记录各管一段，没有打通形成完整的健康管理闭环。

2. **缺乏个性化**：通用助手的建议基于通用知识，没有结合用户的具体健康数据（如血压历史、用药情况、慢性病类型），导致建议不精准。

3. **被动响应**：现有工具大多是被动记录，缺乏主动预警和干预能力。

4. **家庭关怀缺失**：没有考虑到中老年人与子女之间的远程关怀需求，子女无法实时掌握父母健康状况。

5. **交互复杂度**：界面设计针对年轻人，老年人操作困难，缺乏语音交互、大字体等友好设计。

## 三、特色功能设计

### 特色功能：用药依从性追踪与家庭关怀闭环系统

#### 3.1 功能概述

本功能针对中老年人用药管理的核心痛点，构建从用药提醒到依从性追踪，再到子女远程关怀的完整闭环系统。

#### 3.2 业务流程

```
用药提醒触发 → 用户确认服药/未服药 → 记录服药状态 → 计算依从性 → 生成报告 → 子女端推送 → 异常预警
```

#### 3.3 核心逻辑

**① 智能用药提醒**
- 根据用户设定的用药时间，提前10分钟语音提醒
- 支持"已吃"、"没吃"快捷按钮操作
- "没吃"后10分钟自动重提醒

**② 服药状态记录**
- 记录每次服药的时间、药物名称、剂量
- 记录漏服、错服等异常情况
- 支持语音输入确认（如"我吃了"、"还没吃"）

**③ 依从性计算**
- **日依从率**：当日应服药次数 / 实际服药次数 × 100%
- **周依从率**：本周应服药次数 / 实际服药次数 × 100%
- **连续打卡天数**：连续按时服药的天数

**④ 激励机制**
- 连续服药7天：获得"健康小卫士"徽章
- 连续服药30天：获得"健康守护者"徽章
- 每周依从率≥90%：语音表扬和鼓励

**⑤ 异常预警**
- 连续漏服2次：向用户发送关切询问
- 连续漏服3次：自动推送消息至子女端
- 用药时间异常（如比平时晚2小时以上）：主动询问情况

#### 3.4 关键数据

| 数据项 | 说明 | 来源 |
|-------|------|------|
| **用药计划** | 药物名称、剂量、服药时间 | 用户设定 |
| **服药记录** | 服药时间、状态（已吃/没吃） | 用户操作 |
| **依从率** | 日/周/月依从率计算结果 | 系统计算 |
| **徽章成就** | 连续打卡徽章获取情况 | 系统生成 |
| **异常事件** | 漏服、错服、服药时间异常 | 系统检测 |

## 五、与项目其他功能的衔接

### 5.1 与通用能力的衔接

| 特色功能 | 衔接的通用能力 | 衔接方式 |
|---------|---------------|---------|
| **用药提醒** | 意图识别 | 用户说"提醒我吃药"，识别INT_MED_REMIND_SET意图，进入用药链路 |
| **服药确认** | 语音对话 | 用户说"我吃了"或"还没吃"，识别意图并记录状态 |
| **依从性报告** | RAG知识库 | 结合用户用药数据和健康知识库，生成个性化建议 |
| **异常预警** | 用户画像 | 根据用户年龄、慢性病类型，调整预警阈值 |

### 5.2 与已有功能的协同

1. **与个人画像的衔接**：用药计划会记录到用户画像中，健康顾问在给出饮食建议时会考虑用户正在服用的药物，避免食物与药物冲突。

2. **与健康监测的衔接**：血压、血糖异常时，系统会检查用户是否按时服药，为医生诊断提供用药依从性数据。

3. **与子女端的衔接**：子女可以在子女端查看父母的用药依从性报告，了解父母的服药情况，及时进行关怀和提醒。

4. **与语音播报的衔接**：所有提醒和通知都支持语音播报，方便视力不佳或操作困难的老年人使用。

### 5.3 数据流转闭环

```
用户数据（用药记录、健康数据）→ 特色功能处理（依从性计算、异常检测）→ 
生成洞察（报告、预警）→ 通用能力消费（RAG检索、对话回复）→ 
个性化建议 → 用户执行 → 新数据产生 → 闭环
```

## 六、特色功能的独特价值

### 6.1 为什么通用助手做不好？

1. **数据依赖性**：依从性追踪需要用户的真实用药数据，通用助手无法获取这些隐私数据。

2. **家庭协同**：需要子女端和父母端的双向互动，通用助手通常是单人使用的工具。

3. **老年友好**：需要大字体、语音交互、简化操作等专为老年人设计的交互方式。

4. **领域深度**：需要结合医学知识判断用药异常，不是简单的提醒功能。

### 6.2 竞争力分析

| 对比维度 | 通用助手 | 本项目特色功能 |
|---------|---------|---------------|
| **数据来源** | 通用知识 | 用户真实健康数据 |
| **个性化程度** | 通用建议 | 基于用户数据的精准建议 |
| **家庭协作** | 无 | 子女远程关怀闭环 |
| **交互方式** | 通用界面 | 银发友好设计 |
| **预警能力** | 被动查询 | 主动预警干预 |

---

## 七、总结

本项目通过**用药依从性追踪与家庭关怀闭环系统**这一特色功能，解决了中老年人用药管理的核心痛点，构建了从用药提醒到依从性追踪，再到子女远程关怀的完整闭环。这一功能具有以下特点：

1. **人群针对性强**：专为50-80岁中老年人设计，解决他们的切实需求
2. **数据闭环完整**：从数据采集到分析到预警到干预形成完整闭环
3. **家庭协同创新**：打通父母端与子女端，实现远程健康关怀
4. **技术与业务结合**：将意图识别、RAG、语音交互等通用能力与健康管理业务深度融合

这一特色功能不仅满足了用户的表面需求（用药提醒），更深入挖掘了用户的深层需求（健康安全感、子女放心），是通用助手难以实现的差异化价值。
