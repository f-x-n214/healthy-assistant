# 家庭成员绑定功能使用指南

## 功能概述

家庭成员绑定功能允许老年用户将子女或其他家庭成员添加到系统中，实现：
- 健康数据共享（需授权）
- 用药提醒通知
- 紧急情况自动联系
- 漏服事件通知

## 使用方法

### 1. 打开家庭成员管理

点击顶部导航栏的 **"家人"** 按钮（家庭图标），打开家庭成员管理弹窗。

### 2. 添加家庭成员

在弹窗中填写以下信息：

- **关系**：选择家庭成员的关系类型
  - 儿子
  - 女儿
  - 配偶
  - 子女
  - 孙子/外孙
  - 孙女/外孙女

- **称呼（可选）**：可以添加自定义称呼，如"大儿子"、"小女儿"等

- **手机号（可选）**：用于接收通知和紧急联系

点击 **"添加家庭成员"** 按钮完成添加。

### 3. 查看已绑定成员

添加后，家庭成员会显示在"当前已绑定的家庭成员"列表中，格式为：
```
儿子（大儿子） 138****1234
```

### 4. 移除家庭成员

点击每个家庭成员右侧的 **×** 按钮，确认后即可移除。

## 功能说明

### 数据授权

添加家庭成员后，系统会自动提示可以进行数据授权。未来可以实现：
- 子女查看父母健康数据
- 用药记录同步
- 漏服事件通知

### 紧急联系

在紧急情况下，系统可以：
- 自动拨打已绑定的家庭成员电话
- 发送位置信息给家属
- 发送健康异常通知

### 用药提醒

家庭成员可以：
- 接收用药提醒通知
- 查看用药依从性统计
- 收到漏服事件通知

## 技术实现

### 数据存储

家庭成员信息保存在：
- `localStorage.userProfile.familyMembers`
- 格式：`[{relation, label, phone, count}]`

### API 接口

后续可以扩展后端 API：
- `POST /api/family/bind` - 绑定家庭成员
- `DELETE /api/family/unbind` - 解绑家庭成员
- `GET /api/family/members` - 获取家庭成员列表
- `POST /api/family/authorize` - 数据授权

## 示例场景

### 场景 1：绑定儿子
```
1. 点击"家人"按钮
2. 关系选择"儿子"
3. 称呼填写"大儿子"
4. 手机号填写"13812345678"
5. 点击"添加家庭成员"
```

### 场景 2：查看父亲健康数据（子女端）
```
1. 子女登录自己的账号
2. 选择"查看父亲健康数据"
3. 系统验证授权关系
4. 显示父亲的健康记录
```

### 场景 3：用药提醒通知
```
1. 老人设置用药提醒
2. 到时间后系统发送通知给绑定的子女
3. 子女可以查看用药依从性
```

## 注意事项

1. **隐私保护**：健康数据共享需要明确授权
2. **手机号格式**：建议验证手机号格式
3. **多重绑定**：可以绑定多个家庭成员
4. **权限管理**：未来可以支持不同成员的不同权限
