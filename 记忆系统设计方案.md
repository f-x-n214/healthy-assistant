# 银发健康助手 - 记忆系统设计方案

## 一、设计背景

### 为什么需要记忆系统?

**问题: LLM无状态**
- 每次对话独立,无法记住用户信息
- 用户画像缺失,无法个性化服务
- 历史记录缺失,无法统计查询

**解决: 记忆系统**
- 持久化: 数据跨会话可用
- 结构化: 类型安全,易查询
- 自动提取: 从对话中提取数据
- 主动读写: 用户无感知
- 低延迟: 不阻塞响应

### 核心原则

> "记忆有限,想记住就写文件!" - OpenClaw

**记忆系统不是数据库,是认知伙伴。它要像健康管家一样理解你、陪伴你、激励你。**

### 3. 记忆内容如何提取

#### 3.1 数据提取流程

```
用户消息: "我今天晚上吃了阿司匹林一片"
    ↓
意图识别(大模型)
    ↓
返回 extractedData:
{
  "intent": "med_log",
  "confidence": 0.92,
  "extractedData": {
    "drugName": "阿司匹林",
    "dose": { "amount": 1, "unit": "片" },
    "time": { "raw": "今天晚上", "iso": "2026-04-23T20:00:00" },
    "person": "self"
  }
}
    ↓
MemoryService 处理
    ├─ 优先使用 extractedData (来源标记: llm)
    └─ 降级: 正则提取 (来源标记: regex)
    ↓
存储到 medications/2026-04.json
```

#### 3.2 提取策略

**优先级1: 大模型提取**
- 准确率高,支持复杂表达
- 自动标准化时间、单位
- 来源标记: `llm`

**优先级2: 正则降级**
- 大模型失败时降级
- 处理简单格式
- 来源标记: `regex`

**示例: 正则降级**
```javascript
// 大模型提取失败,使用正则
function extractMedicationByRegex(utterance) {
  const patterns = [
    /吃了(.+?)(\d+)(片|粒|包)/,
    /服用(.+?)(\d+)(片|粒|包)/
  ];
  
  for (const pattern of patterns) {
    const match = utterance.match(pattern);
    if (match) {
      return {
        drugName: match[1],
        dose: { amount: Number(match[2]), unit: match[3] },
        source: 'regex'
      };
    }
  }
  return null;
}
```

#### 4.2 缓存策略

```javascript
class MemoryCache {
  constructor() {
    this.cache = new Map();
    this.ttl = {
      profile: Infinity,        // 会话级
      records: 5 * 60 * 1000,   // 5分钟
      statistics: 10 * 60 * 1000 // 10分钟
    };
  }
  
  get(key, type) {
    const item = this.cache.get(key);
    if (!item) return null;
    
    const age = Date.now() - item.timestamp;
    if (age > this.ttl[type]) {
      this.cache.delete(key);
      return null;
    }
    
    return item.data;
  }
  
  set(key, data, type) {
    this.cache.set(key, {
      data,
      timestamp: Date.now(),
      type
    });
  }
}
```

### 5. 记忆何时更新

#### 5.1 更新时机

| 类型 | 时机 | 方式 | 优先级 |
|-----|-----|------|--------|
| 用药记录 | 用户报告用药 | 即时同步 | 高 |
| 血压记录 | 用户报告血压 | 即时同步 | 高 |
| 提醒设置 | 用户设置提醒 | 即时同步 | 高 |
| 里程碑 | 检测到里程碑 | 后台异步 | 中 |
| 阶段转换 | 使用频率变化 | 后台异步 | 中 |
| 统计摘要 | 每日定时 | 定期更新 | 低 |

#### 5.2 异步更新优势

- **不阻塞用户响应**: 用户体验优先
- **失败可重试**: 数据可靠性
- **数据来源可追溯**: 审计需求

```javascript
// 异步更新队列
class UpdateQueue {
  constructor() {
    this.queue = [];
    this.processing = false;
  }
  
  enqueue(task) {
    this.queue.push(task);
    if (!this.processing) {
      this.process();
    }
  }
  
  async process() {
    this.processing = true;
    while (this.queue.length > 0) {
      const task = this.queue.shift();
      try {
        await task.execute();
      } catch (error) {
        console.error('更新失败:', error);
        // 重试逻辑
        if (task.retries < 3) {
          task.retries++;
          this.queue.push(task);
        }
      }
    }
    this.processing = false;
  }
}
```

## 三、记忆系统实现方案

### 1. MemoryService 核心类

```javascript
class MemoryService {
  constructor(userId) {
    this.userId = userId;
    this.cache = new MemoryCache();
    this.updateQueue = new UpdateQueue();
    this.basePath = `data/users/${userId}`;
  }
  
  // 加载用户画像
  async loadProfile() {
    const cached = this.cache.get('profile', 'profile');
    if (cached) return cached;
    
    const profile = await this.readFile(`${this.basePath}/profile.md`);
    this.cache.set('profile', profile, 'profile');
    return profile;
  }
  
  // 加载核心记忆
  async loadMemory() {
    const cached = this.cache.get('memory', 'profile');
    if (cached) return cached;
    
    const memory = await this.readFile(`${this.basePath}/MEMORY.md`);
    this.cache.set('memory', memory, 'profile');
    return memory;
  }
  
  // 保存用药记录
  async saveMedication(data) {
    const month = this.getCurrentMonth();
    const filePath = `${this.basePath}/medications/${month}.json`;
    
    let records = await this.readFile(filePath) || { records: [] };
    records.records.push({
      id: this.generateId('med'),
      ...data,
      createdAt: new Date().toISOString()
    });
    
    await this.writeFile(filePath, records);
    
    // 异步更新统计
    this.updateQueue.enqueue({
      execute: () => this.updateMedicationStats(month),
      retries: 0
    });
  }
  
  // 保存血压记录
  async saveBloodPressure(data) {
    const month = this.getCurrentMonth();
    const filePath = `${this.basePath}/health/blood-pressure-${month}.json`;
    
    let records = await this.readFile(filePath) || { records: [] };
    records.records.push({
      id: this.generateId('bp'),
      ...data,
      createdAt: new Date().toISOString()
    });
    
    await this.writeFile(filePath, records);
    
    // 异步更新统计和检测里程碑
    this.updateQueue.enqueue({
      execute: () => this.updateBPStats(month),
      retries: 0
    });
    this.updateQueue.enqueue({
      execute: () => this.detectMilestone(),
      retries: 0
    });
  }
  
  // 查询最近记录
  async queryRecentRecords(type, days = 7) {
    const cacheKey = `recent_${type}_${days}`;
    const cached = this.cache.get(cacheKey, 'records');
    if (cached) return cached;
    
    const records = await this.loadRecordsByDays(type, days);
    this.cache.set(cacheKey, records, 'records');
    return records;
  }
  
  // 检测里程碑
  async detectMilestone() {
    const profile = await this.loadProfile();
    const medications = await this.queryRecentRecords('medication', 30);
    const bpRecords = await this.queryRecentRecords('blood_pressure', 30);
    
    for (const rule of MILESTONE_RULES) {
      if (rule.condition({ medications, bpRecords, profile })) {
        await this.updateMilestone(rule);
        return rule;
      }
    }
    return null;
  }
  
  // 更新里程碑
  async updateMilestone(milestone) {
    const memory = await this.loadMemory();
    memory.milestones = memory.milestones || [];
    memory.milestones.push({
      ...milestone,
      date: new Date().toISOString().split('T')[0]
    });
    
    await this.writeFile(`${this.basePath}/MEMORY.md`, memory);
  }
  
  // 辅助方法
  getCurrentMonth() {
    const now = new Date();
    return `${now.getFullYear()}-${String(now.getMonth() + 1).padStart(2, '0')}`;
  }
  
  generateId(prefix) {
    const timestamp = new Date().toISOString().replace(/[-:T]/g, '').slice(0, 15);
    return `${prefix}_${timestamp}_${Math.random().toString(36).slice(2, 5)}`;
  }
}
```

const MILESTONE_RULES = [
  {

    },

      const adherence = calculateAdherence(medications);
      return adherence >= 0.9;

// 辅助函数
function checkContinuousDays(records, days) {
  if (records.length < days) return false;
  
  const dates = [...new Set(records.map(r => 
    new Date(r.time).toISOString().split('T')[0]
  ))].sort().reverse();
  
  for (let i = 0; i < days - 1; i++) {
    const current = new Date(dates[i]);
    const next = new Date(dates[i + 1]);
    const diff = (current - next) / (1000 * 60 * 60 * 24);
    if (diff !== 1) return false;
  }
  return true;
}

function calculateAdherence(medications) {
  // 简化计算:实际记录数 / 预期记录数
  const days = 7;
  const expectedPerDay = 2; // 假设每天2次用药
  const expected = days * expectedPerDay;
  const actual = medications.filter(m => {
    const daysAgo = (Date.now() - new Date(m.time)) / (1000 * 60 * 60 * 24);
    return daysAgo <= days;
  }).length;
  
  return Math.min(actual / expected, 1);
}

function average(arr) {
  return arr.reduce((a, b) => a + b, 0) / arr.length;
}

function generateReply({ utterance, ai, safety, profile, memory, milestone }) {
  let reply = '';
  
  // 基础回复
  switch (ai.intent) {
    case INTENTS.MED_ADD:
      reply = `好的,已记录您服用了${ai.extractedData.drugName}。`;
      break;
    case INTENTS.BP_ADD:
      const { systolic, diastolic } = ai.extractedData;
      reply = `已记录血压${systolic}/${diastolic}mmHg。`;
      
      // 个性化建议
      if (systolic > 140 && profile.chronicDiseases.includes('高血压')) {
        reply += '\n血压偏高,建议按时服药,保持低盐饮食。';
      }
      break;
    // 其他意图...
  }
  
  // 里程碑激励
  if (milestone) {
    reply += `\n\n${milestone.message}`;
  }
  
  return reply;
}

## 六、常见陷阱与解决方案

| 陷阱 | 解决方案 |
|-----|---------|
| 每次重新提取 | 意图识别时同步提取,避免二次处理 |
| 所有数据一个文件 | 按月分文件,避免单文件过大 |
| 没有降级方案 | 正则降级,确保大模型失败时仍可工作 |
| 同步更新阻塞 | 异步更新队列,不阻塞用户响应 |
| 缓存过期未清理 | 设置TTL,定期清理过期缓存 |
| 文件损坏无备份 | 定期备份,支持数据恢复 |
