幻灯片 15:
8.1🤖 CodeArts 为什么没有长期记忆？
3. CodeArts 设计理念                 
  "每次对话都是独立的、全新的开始"         
  优点：                                  
  - 隐私保护：不存储用户数据              
  - 无状态：易于扩展和负载均衡            
  - 安全性：无数据泄露风险                

  缺点：                                  
  - 无法记住用户偏好                      
  - 无法积累项目知识                      
  - 每次需要重新了解上下文
4. 替代方案：项目文件作为"记忆"
用户项目文件 = CodeArts 的"长期记忆"
── README.md        	→ 项目说明
── docs/               	→ 项目文档
   └── progress.md     	→ 进度记录
── .cursorrules       	 → 项目规则
└── package.json       	 → 项目配置
用户: "继续上次的工作"
    ↓
CodeArts: 读取 docs/progress.md
    ↓
CodeArts: "好的，我看到上次完成了..."
    ↓
继续工作

⚡ 记错，也要记对
 "Record from failure AND success. If you only save corrections, you will avoid past mistakes but drift away from approaches the user has already validated, and may grow overly cautious."
翻译：记错误，也要记认可。只记纠正的话，AI会越来越缩手缩脚。

幻灯片 24:
⚡ 默认认可
 我纠正它，它记住了，下次不犯。
你说"这个方向对"，它要记。
你不说话但没有推翻它的方案，它也要记——这是一个默认认可的信号。
"Corrections are easy to notice; confirmations are quieter — watch for them."

幻灯片 25:
⚡为什么要记Why？ 
反馈记忆三个部分：规则本身、Why（当时的理由）、How to apply（什么场景触发）。
"Knowing why lets you judge edge cases instead of blindly following the rule." 
知道原因才能灵活应用边界情况，不知道原因就只能机械执行。

幻灯片 26:
8-3.Claude Code记忆系统
3.文件系统——不是向量数据库
很多人会猜：Claude Code这套记忆系统，底层一定用了向量数据库。但没有——纯文件系统。
每条记忆是一个独立的Markdown文件，人可以直接打开看，可以手动改，可以用git管理，可以diff，可以回滚，可以grep。
MEMORY.md是索引文件，最多200行，每条记忆不超过150字，超了会截断，还会自动提示AI"索引太长了，下次写精简一点"。
为什么不用向量数据库？源码里没解释。我的理解是：可以被人类审查的记忆，信任成本低得多。
用向量数据库，你不知道AI记住了什么。用Markdown文件，你打开就看到了。你知道你的AI在想什么、记着什么——这本身就是一种控制感。AI的控制感从哪来？就是你能用自然语言描述它的记忆，随时打开、随时修改。

幻灯片 30:
5.2 怎么记 - 文件格式选择
记忆文件格式
- Markdown 优点：人可读、可编辑、可版本控制、可 diff
- JSON 优点：结构化、易解析、类型安全
核心原则
数据特性	选择格式	原因	
人需要看的	Markdown	可读、可编辑	
程序需要解析的	JSON	结构化、类型安全	
本项目选择
profile.md → 用户画像（人可读）
MEMORY.md → 核心记忆（人可读）
exercises/*.json → 运动记录（程序统计）
diet/*.json → 饮食记录（程序统计）
