# 从项目到技能：包装发布实战教程

**目标**：教你把自己的项目内容提取出来，包装成标准技能（Skill），发布到 SKILLS.SH / agentskills.io / CodeArts 技能平台，让其他用户安装使用。

**为什么做这件事？**
- 把项目从"作业"变成"作品"——有真实用户安装使用的技能
- 简历加分：发布过技能包 > 做过课程作业
- 反向驱动质量：要发布就要代码规范、文档完善、测试通过，这本身就是质量提升
- 社区贡献：你的领域知识可以帮到其他人

---

## 一、技能是什么？— 用官方技能 `pdf` 作为例子

### 1.1 一个技能长什么样？

以官方发布的 `pdf` 技能为例（`E:\qclaw-essential-skills\pdf`）：

```
pdf/                          ← 技能根目录（必须与 name 一致）
├── SKILL.md                  ← 核心入口：YAML元数据 + 使用指南
├── forms.md                  ← 补充文档：表单填写专项指南
├── reference.md              ← 补充文档：高级参考
├── LICENSE.txt               ← 许可证
└── scripts/                  ← 可执行脚本
    ├── setup_chinese_pdf.py  ← CJK字体引导工具
    ├── check_fillable_fields.py
    ├── fill_fillable_fields.py
    └── ...（9个脚本）
```

### 1.2 SKILL.md 是灵魂

```yaml
---
name: pdf                              # 必需：kebab-case，与目录名一致
description: >                         # 必需：1-1024字符，含触发关键词
  Use this skill whenever the user wants 
  to do anything with PDF files...
license: Proprietary. LICENSE.txt has complete terms
---

# PDF Skill

Overview、Quick Start、各工具用法、规则、参考...
```

**关键理解**：

| 要素 | 作用 |
|------|------|
| `name` | Agent 匹配技能的标识，必须与目录名一致 |
| `description` | Agent 判断"何时触发"的依据，必须包含触发关键词 |
| SKILL.md 主体 | Agent 触发后加载的使用指南（<500行推荐） |
| 补充文档 | 按需加载的深度知识（references/、forms.md 等） |
| scripts/ | 可执行工具脚本，Agent 可调用 |

### 1.3 三级渐进披露

| 级别 | 内容 | 何时加载 | 大小 |
|------|------|---------|------|
| L1 | name + description | 始终在上下文 | ~100词 |
| L2 | SKILL.md 主体 | 技能触发时 | <5k词 |
| L3 | scripts/references/knowledge | Agent 按需使用 | 不限 |

> pdf 技能的 L1 = "PDF files相关就触发"，L2 = SKILL.md 中的操作指南，L3 = forms.md + reference.md + 9个脚本

## 二、动手包装：5步把项目内容变成技能

### Step 1：识别可提取的内容

问自己3个问题：

1. **我的项目有哪些知识/数据是通用的？** — 不只给我自己用，别人也能受益的
2. **有哪些工具函数/脚本可以独立使用？** — 不依赖我的完整项目就能跑的
3. **有哪些最佳实践/规则值得分享？** — 我踩过的坑、总结的模式

**以我们项目为例**：
| 可提取内容 | 类型 | 说明 |
|-----------|------|------|
| 50+女性运动安全知识库（43模块） | knowledge/ | 领域数据，别人做健康应用直接用 |
| 中医九种体质+六维养生 | knowledge/ | 独立知识域 |
| 中国食物成分数据（1755条） | knowledge/ | 纯数据资产 |
| 运动安全检查逻辑 | scripts/ | 可独立调用的Python脚本 |
| 体质辨识流程 | scripts/ + references/ | 代码+指南 |

### Step 2：确定技能架构

**两种架构选一种**：

| 架构 | 适用场景 | 例子 |
|------|---------|------|
| **单技能** | 内容聚焦一个领域 | pdf（PDF处理） |
| **技能集（双层）** | 多个独立知识域 + 跨域聚合 | science-based-lifter、fiftyplus-health-kb |

**判断标准**：如果你的知识能清晰分成3+个独立领域，且每个域都有独立价值 → 用技能集；否则 → 单技能。

### Step 3：创建标准目录结构

**单技能标准结构**：

```
your-skill-name/
├── SKILL.md              ← 必需
├── README.md             ← 必需
├── CHANGELOG.md          ← 必需
├── LICENSE.txt           ← 必需（推荐 Apache-2.0）
├── _meta.json            ← 必需：{"version":"1.0.0"}
├── _meta-zh.json         ← CodeArts 市场必需
├── scripts/              ← 可选：可执行脚本
├── references/           ← 可选：深度参考文档
├── knowledge/            ← 可选：知识库数据
├── evals/                ← 推荐：评估测试
│   └── evals.json
└── tests/                ← 推荐：开发者测试
```

**技能集结构**（多技能）：

```
skill-collection/
├── skills-lock.json      ← 版本锁
├── domain-skill-1/       ← 领域技能1
│   ├── SKILL.md
│   ├── references/
│   └── evals/
├── domain-skill-2/       ← 领域技能2
└── meta-skill/           ← 元技能（跨域聚合）
```

### Step 4：编写 SKILL.md

**模板**：

```yaml
---
name: your-skill-name            # kebab-case，与目录名一致
description: >                   # 1-1024字符，必须包含触发关键词
  简述技能做什么、何时使用。
  嵌入触发关键词：xxx查询、yyy推荐、zzz检查。
  当用户需要AAA、BBB、CCC时触发此技能。
metadata:
  version: 1.0.0
compatibility: >                 # 可选
  Python 3.10+（可选：依赖列表）
---

# Your Skill Name

一句话说明。

## 快速开始

最简单的使用方式（3行以内）。

## 核心功能

| 功能 | 说明 | 使用方式 |
|------|------|---------|
| 功能1 | ... | 命令/调用方式 |

## 参考文档

- [文档1](references/xxx.md)
- [文档2](references/yyy.md)
```

**写好 description 的技巧**：
- 必须包含用户可能说的关键词（Agent 靠这个匹配触发）
- 用"当用户需要...时触发此技能"的句式
- 覆盖主要使用场景

### Step 5：编写辅助文件

#### README.md

面向人类读者的说明，包含：安装方式、功能列表、使用示例、开发指南。

#### _meta.json

```json
{"version": "1.0.0"}
```

#### _meta-zh.json（CodeArts 市场）

```json
{
  "owner": "your-team",
  "slug": "your-skill-name",
  "displayName": "中文显示名",
  "brief": "一句话简介",

  "labels": ["标签1", "标签2"],
  "license": "Apache-2.0",
  "type": "SOP",
  "category": "health-wellness",
  "lifecycleState": "PRODUCTION",
```

## 五、常见问题

### Q1：我的项目必须做成技能吗？

不是必须，但强烈推荐。做技能的过程就是质量提升的过程：
- 写 SKILL.md → 梳理功能、明确触发场景
- 写 evals.json → 验证核心功能是否正确
- 写 references/ → 沉淀领域知识
- 规范目录结构 → 代码组织更清晰

### Q2：知识库数据用 JSON 还是 Markdown？

| 格式 | 适用场景 | 例子 |
|------|---------|------|
| JSON | 结构化数据、可被脚本解析 | 运动数据库、食物成分表 |
| Markdown | 叙述性知识、可被Agent直接引用 | 安全规则说明、体质指南 |

两者可以共存：`knowledge/` 放 JSON，`references/` 放 Markdown。

### Q3：scripts/ 是必需的吗？

不是。参考 pdf 技能，scripts/ 是可选的增强。如果你的技能主要是知识库（Agent 直接加载），可以不提供脚本。脚本的价值在于提供精确的查询/计算能力（如安全检查、营养计算）。

### Q4：技能的 name 怎么取？

- 只允许小写字母、数字、连字符
- 不能以连字符开头/结尾，不能有连续连字符
- 必须与 SKILL.md 所在目录名完全一致
- 好的名字：`fitness-kb`、`pdf`、`exercise-safety-50plus`
- 坏的名字：`FitnessKB`、`fitness_kb`、`50plus-health`
